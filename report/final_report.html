Top 4 recommendations

P0

Ground actions in caller-confirmed facts

Owner: Dialog/NLU + prompt engineering

Evidence: 21 calls (10.5%) — CONTEXT_IGNORED ×15, WRONG_GUIDANCE ×6 (critical). Signature: negation inversion — caller's “No” processed as a yes (028, 035); countdown advice after a confirmed permanent lockout (065).

Approach: polarity-check every confirmation {affirm/deny/unclear} before branching; persist confirmed facts (lockout type, notice status) as slots that hard-gate guidance; regression suite from 028/035/065/073.

Accept: zero negation inversions on golden set; combined code rate ≤ 1.5% on corpus re-run.
P1

Make satisfied dialog states idempotent

Owner: Voice-agent flow/state-machine engineering

Evidence: STATE_ECHO in 68 calls (34%) — the dominant defect. Consent re-asked after a clean “Yes” (040, 042, 099); identity gate re-fired (044); interrupted messages restarted from the top (043).

Approach: slot-filled guards so satisfied states can’t re-fire; commit affirmatives on first parse; resume — never restart — interrupted utterances on barge-in.

Accept: STATE_ECHO ≤ 10% of calls on corpus re-run; zero double consent-asks on golden set.
P1

Enforce the identity gate before any action

Owner: Voice-agent flow engineering

Evidence: 10 calls (5%) fail identity (critical): ID_SKIPPED ×7 (acted before asking a name — 084, 111), ID_INVALID ×3 (non-name accepted — 066, 146). The correct behavior already exists in production: 039 queried a garbled answer and re-asked.

Approach: make name-capture a hard precondition of action states (link/transfer/callback unreachable while unset); add a name-plausibility validator with one re-ask, per 039.

Accept: identity fail rate 0% on corpus re-run.
P1 NEW — containment analysis

Stop unsolicited transfer nudges; close resolved calls

Owner: Voice-agent flow/prompt engineering

Evidence: dedicated Pass C over all 87 transfer-offer calls: 11 flagged (5.5%), incl. 7 containable transfers (024, 051, 092, 122, 149, 180, 197). Signature: “Would you like to speak with customer support as well?” appended after the need was resolved (7 of 11). 041 vs 149: identical resolved flows — 041’s caller said “No. Thank you.”, 149’s said yes and became a transfer. In 154/197 the human’s post-transfer answer was the 7-day grace policy Mia already knows.

Approach: gate transfer offers on (a) caller asked, (b) CANNOT-zone need, or (c) designed flow; resolved state goes straight to farewell — delete the trailing nudge; require one containment attempt on answerable policy questions before any escalation offer.

Accept: flagged calls ≤ 2 on corpus re-run (scripts/containment.py); transferred share drops, failed doesn’t rise.
Watch items: CB_NO_CONFIRM hits 15.4% of callback flows (the safety-net state) — folds into P0’s polarity work · MULTI_Q in 23 calls (style-guide fix) · CUT_TRANSCRIPT 14% is a platform/telephony recording issue, not an agent defect.
Method: Pass A literal fact extraction → Pass B rubric judge → Pass C transfer-containment classifier (Claude headless, 8 workers), schema-validated with retry + bracket repair; evidence quotes substring-verified against transcripts; checkpointed reruns. Judge spend ≈ $262.
